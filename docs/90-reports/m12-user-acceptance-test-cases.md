# 用户验收测试用例

> 本文档来自 M12 内测验收基线，当前保留为历史测试参考。统一的当前验收基线将在后续文档规范阶段重新整理。

## 测试前置条件

- 在开始正式内测验收前运行 `scripts/reset-m12-test-data.ps1`，确保本地只保留 `admin`、`m12_alice`、`m12_bob` 三个测试账号。
- 普通功能修复和开发验证期间不要求每次重置数据；只有需要重新建立干净验收基线时才执行。
- 启动本地依赖、后端和前端，访问 `http://127.0.0.1:5173/login`。
- 使用三种角色分别验证：管理员 `admin`、成员 `m12_alice`、成员 `m12_bob`。
- 每条用例记录：通过、失败、阻塞，以及必要截图或接口错误信息。

## 用例清单

| 任务 | 验收目标 | 手工步骤 | 期望结果 |
| --- | --- | --- | --- |
| M12-T01 | 内测入口可用 | 用三个账号分别登录、刷新页面、退出登录 | 登录成功；刷新后会话保持；退出后回到登录页 |
| M12-T02 | 工作台和导航可用 | 登录后检查左侧入口，依次进入 IM、项目、文档、表格、审批、搜索 | 入口清晰；页面无空白崩溃；空状态能指导下一步操作 |
| M12-T03 | IM 核心链路可用 | 创建或进入会话，发送普通消息、@成员、包含业务链接的消息 | 消息按时间展示；@提醒可见；业务链接显示标题和摘要卡片 |
| M12-T04 | 项目与 Bug 流程可用 | 创建项目，创建 Bug，设置负责人和优先级，变更状态，添加评论 | 列表、详情、状态和评论同步更新；成员账号能查看相关内容 |
| M12-T05 | 文档与表格入口可用 | 创建文档和表格，在 IM 中分享链接，从卡片跳转回对象详情 | 链接卡片可识别对象类型；跳转后内容可读；返回 IM 不丢上下文 |
| M12-T06 | 通知与未读可用 | 触发 @、评论、状态变更，查看通知中心和未读计数 | 通知产生及时；点击通知能进入来源对象；已读状态稳定 |
| M12-T07 | 审批入口可用 | 发起一个审批，切换到审批人处理，再查看申请人状态 | 待办、审批动作、审批结果在双方视角一致 |
| M12-T08 | 搜索可用 | 搜索消息关键字、项目标题、文档标题、表格名称 | 搜索结果包含正确类型、标题、摘要和跳转链接 |
| M12-T09 | 权限和异常状态可理解 | 用成员账号访问管理入口或不存在的对象链接 | 权限不足、对象不存在等状态有明确提示，不出现白屏 |
| M12-T10 | 测试数据可重置 | 完成若干新增数据后重新运行重置脚本 | 业务数据被清空；三个测试账号仍可登录 |
| M12-T11 | 关键路径回归 | 按 T01-T09 快速重走一遍主链路 | 没有阻塞内测的回归问题 |
| M12-T12 | 内测试运行准备 | 记录发现的问题、严重级别、复现步骤和期望修复优先级 | 问题可复现、可分派；无 P0/P1 阻塞后进入小范围试用 |

## 通过标准

- P0/P1 问题为 0。
- P2 问题有明确绕行或已进入修复队列。
- 三个保留账号均可登录，且重置脚本可重复执行。
- IM、项目、文档、表格、通知、搜索至少完成一轮跨模块闭环。
