# AI 核验测试用例

> 本文档来自 M12 AI 核验基线，当前保留为历史测试参考。统一的当前 AI 核验基线将在后续文档规范阶段重新整理。

## 核验原则

- 只有在明确进入 M12 内测基线、用户通知需要重置，或本轮任务包含 M12-T01 数据基线时，才运行 `scripts/reset-m12-test-data.ps1`。
- 自动核验优先覆盖可重复的接口、数据库约束、构建、测试和浏览器冒烟。
- 手工体验问题不伪装成自动通过；无法自动判断的部分记录为人工待验。
- 自动测试可能产生临时数据，若本轮最终目标是交付干净内测环境，应在全部测试完成后再执行一次 M12 重置。

## 基线检查

| 检查项 | AI 执行动作 | 期望结果 | 证据 |
| --- | --- | --- | --- |
| 数据重置 | 在用户明确要求时运行 `scripts/reset-m12-test-data.ps1` | 脚本成功；输出三条保留账号 | 终端输出和 `.local-reports/pre-m12-cleanup-*.sql` |
| 账号数量 | 查询 `users` | 仅有 `admin`、`m12_alice`、`m12_bob` 三个账号 | SQL 查询结果 |
| 账号登录 | 调用 `/api/auth/login` | 三个账号均返回访问令牌 | API 响应状态 |
| 质量门禁 | 运行 `pnpm verify`，必要时运行 `pnpm verify:full` | 后端测试、前端 lint/build、chunk budget、懒加载、敏感信息扫描通过 | 质量门禁报告 |

## 任务核验清单

| 任务 | AI 核验动作 | 期望结果 |
| --- | --- | --- |
| M12-T01 | API 登录三个账号；浏览器打开登录页和登录后首页 | 登录、刷新、退出链路无异常 |
| M12-T02 | 浏览器访问主导航页面：`/im`、`/projects`、`/docs`、`/bases`、`/approvals`、`/search` | 页面渲染非空；控制台无阻塞错误 |
| M12-T03 | 通过 API 创建会话并发送普通消息、@消息、业务链接消息 | 消息列表顺序正确；链接解析字段存在 |
| M12-T04 | 通过 API 创建项目和 Bug，更新状态并评论 | 列表和详情接口返回一致数据 |
| M12-T05 | 通过 API 创建文档和表格，向 IM 发送对应链接 | 对象可读取；链接卡片能返回类型、标题、摘要 |
| M12-T06 | 触发 @、评论、状态变更后查询通知接口 | 通知存在；未读状态可变更为已读 |
| M12-T07 | 创建审批实例，切换审批人处理 | 待办和实例状态符合审批流 |
| M12-T08 | 创建可搜索对象后调用搜索接口 | 结果包含对象类型、标题、摘要、跳转地址 |
| M12-T09 | 用成员账号访问管理接口和不存在对象 | 返回 403 或 404；前端显示可理解状态 |
| M12-T10 | 在产生业务数据后再次运行重置脚本 | 业务表清空；保留账号仍存在 |
| M12-T11 | 运行关键路径冒烟脚本或浏览器回归 | 核心页面无白屏；主要接口无 5xx |
| M12-T12 | 生成本轮核验记录和缺陷清单 | 记录包含复现步骤、影响范围、建议优先级 |

## 补充约束

- 不要在每次普通代码验证后自动重置数据；只有用户要求、M12-T01、或准备手工验收基线时才执行。
- 如果执行过会写入本地库的集成测试，并且本轮目标是交付干净内测环境，应在最后执行一次 M12 重置。
- 发现表结构新增后，如果重置脚本因外键失败而停止，应先补充清理范围，再继续 M12。
- 链接预览、协同编辑、权限和通知属于跨模块风险点，出现失败时优先保留接口响应、数据库记录和浏览器截图。
