# 历史执行报告索引

`90-reports/` 中的文件全部是里程碑执行时点的历史证据，不是当前产品、架构、API 或数据库事实来源。

读取规则：

- 当前事实优先读取 `docs/README.md` 列出的 active 文档和当前代码。
- 仅在排查某个里程碑的实现过程、验证命令或历史决策时读取对应报告。
- 文件名 `m1` 到 `m12` 曾被多个专题复用，必须以报告 front matter 的 `title`、`milestone` 和正文范围识别，不能只按文件名推断。
- `completed`、`active` 等历史状态在本目录统一按 archived 语义理解。
- 报告中的 `/docs`、`Document*`、旧表名、旧测试数和旧路由均可能已被后续迁移替代。

主要专题前缀：

| 前缀 | 历史范围 |
| --- | --- |
| `kb-name-*` | 知识库唯一命名和旧文档兼容清理 |
| `ui-split-*` 或标题中的 `UI-SPLIT-*` | 用户工作台与管理后台拆分 |
| 标题中的 `KB-UX-*` | 知识库内容体验与块编辑器 |
| `m25-*` 到 `m50-*` | 平台能力、协作、知识内容底座和试运行阶段 |

新执行报告必须使用完整专题里程碑名，避免再次生成含糊的裸 `mX-execution-report.md`。
