# 当前路线图

本文档是唯一活跃路线图。已完成的 M25-M30 计划已归档到 `docs/99-archive/superseded-roadmaps/m25-m30-completed-roadmap.md`。M31 已形成可重复运行的多角色协同仿真基线。M32-M40 是下一阶段从“模块可用”走向“团队真实可用”的当前执行计划。

## 当前阶段判断

系统已经完成基础模块、核心协作体验打磨、搜索通知权限审计横向补强、Base 深度协同能力、多端体验基础、工程治理可观测补强和单节点交付运维收口，并已建立 M31 多角色协同仿真基线。M32 已补齐文档团队空间、树形目录、路径、排序、移动和归档恢复基础；M33 已补齐普通表格块和 Base/事项/消息等平台对象嵌入块；M36 已完成通知处理、中文搜索、权限解释和审计页面第一轮统一；M37 已完成 Base 字段校验、记录详情、对象关系、视图显隐、文档内 Base 视图预览和 CSV 导入导出；M38 已完成移动 Web 响应式 Shell、核心页面窄屏降级、PWA 基础和桌面薄壳边界记录；M39 已完成测试库隔离、性能基线、请求日志、健康指标、备份恢复 dry-run、安全审计门禁和 CI/本地门禁对齐；M40 已完成小团队真实试运行准入清单、10 人角色剧本、初始化模板、问题分级、迁移清理策略、管理员运维手册、完整回归和 Go/No-Go 报告。当前可以进入受控小团队真实试运行执行阶段，但试运行反馈和 P0/P1 修复闭环尚未发生。

当前执行重点：

1. M31 基线已覆盖 10 人、5 项目、多分支需求/BUG 流转。
2. M31 通过 `pnpm sim:m31` 与 `pnpm smoke:m31` 保留为可显式触发的跨模块协同回归工具，用于验证 IM、项目、文档、Base、通知、权限、审计和搜索的联动质量。
3. M32-M40 已按单里程碑、单闭环推进；M40 作为试运行准入专项已显式执行 `pnpm verify:full`、`pnpm data:reset` 和 `pnpm smoke:m31`。后续日常迭代仍不自动重置 M31 数据或运行 M31 全链路回归，除非任务本身或用户明确要求。

## M31 分支覆盖

需求分支：

- 正常确认并拆任务。
- 需求信息不足，打回产品补充。
- 需求中途变更，产生文档版本和 IM 讨论。
- 需求延期，通知相关成员。
- 需求取消，状态、审计、通知正确。

BUG 分支：

- BUG 有效，进入修复。
- BUG 信息不足，打回补充。
- BUG 重复，关联已有 BUG。
- BUG 无法复现，关闭或待确认。
- 修复后验证失败，打回开发。
- 二次修复后验证通过并关闭。
- 无权限用户访问被拦截。

## M32-M40 里程碑任务计划
