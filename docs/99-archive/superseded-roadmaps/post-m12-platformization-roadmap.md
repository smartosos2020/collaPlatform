# M12 后平台化里程碑任务计划

## 1. 目标

M12 之后，系统不再只按单个业务模块堆功能，而是进入 Lark 形态所需的平台化补强阶段。

核心目标：

- 保持模块化单体，不急于拆微服务。
- 用统一对象、权限、实时、搜索和通知把 IM、项目、文档、表格、审批串成一个协作平台。
- Web 继续优先，但 API、实时协议、深链和对象模型要为桌面端、移动端预留。
- 每个里程碑都必须有可运行交付物、测试路径和验收标准。

## 2. 执行顺序

| 顺序 | 里程碑 | 目标 |
| --- | --- | --- |
| 1 | M12 现有能力稳定化 | 让当前 IM、项目、文档、表格、通知、搜索达到内测准入状态 |
| 2 | M13 平台对象模型强化 | 所有核心业务对象可引用、可预览、可跳转、可搜索 |
| 3 | M14 统一权限决策层 | 搜索、卡片、详情接口使用一致权限判断 |
| 4 | M15 统一实时通道 | IM WebSocket 升级为平台级 realtime gateway |
| 5 | M16 文档模块升级一期 | 从整文 Markdown 保存升级到块模型基础 |
| 6 | M17 搜索平台化 | 从模块搜索升级为权限感知的全局搜索 |
| 7 | M18 多端基础架构 | 为 Web、桌面端、移动端共用能力打基础 |

## 3. M12 现有能力稳定化

目标：把当前系统打到可持续内测状态，先稳定已有闭环。

| 编号 | 任务 | 交付物 | 验收标准 |
| --- | --- | --- | --- |
| M12-T01 | 清理并固化测试数据重置流程 | `scripts/reset-m12-test-data.ps1` 和脚本说明 | 可重复执行；保留 `admin`、`m12_alice`、`m12_bob` |
| M12-T02 | 生成用户验收和 AI 核验用例 | `docs/m12-user-acceptance-test-cases.md`、`docs/m12-ai-verification-test-cases.md` | 用例覆盖 IM、项目、文档、表格、审批、搜索、通知 |
| M12-T03 | IM 核心交互稳定化 | 单聊、群聊、右键菜单、已读、撤回、置顶、表情 | 浏览器冒烟和集成测试通过 |
| M12-T04 | 跨模块端到端回归 | IM 分享事项、文档、表格链接并显示卡片 | 链接卡片、跳转、权限态一致 |
| M12-T05 | 修复明显体验问题 | 当前阻塞内测的问题清单清零 | 无 P0/P1 阻塞问题 |
| M12-T06 | 沉淀高频测试脚本 | `scripts/README.md` 中记录重置、质量门禁、审计快照 | 新一轮 AI 工作能按脚本执行 |
| M12-T07 | 确认内测账号和基础数据 | 三个账号及密码记录 | 三个账号均可登录 |
| M12-T08 | 输出内测准入清单 | 本文档和 M12 测试文档形成准入依据 | 可判断是否进入小范围试用 |

通过标准：

- 本地依赖、后端、前端均可启动。
- `pnpm verify` 或等价质量门禁通过。
- 三个 M12 测试账号可登录。
- 核心模块无白屏、无阻塞级错误。
- 业务测试数据可按需重置，不依赖人工 SQL。

## 4. M13 平台对象模型强化

目标：让文档、事项、表格、消息、审批、文件都成为统一可引用对象。

| 编号 | 任务 | 交付物 | 验收标准 |
| --- | --- | --- | --- |
| M13-T01 | 盘点现有 `object_links` 和 resolver | 对象类型清单 | 明确已接入和缺口 |
| M13-T02 | 定义统一对象摘要结构 | 标题、摘要、图标、路径、权限状态 | 所有卡片字段一致 |
| M13-T03 | 补齐核心对象 resolver | issue、document、base、base_table、base_record、message、approval | 对象摘要接口覆盖核心对象 |
| M13-T04 | 统一内部链接解析规则 | `/issues/{id}`、`/docs/{id}`、`/bases/{id}`、`colla://...` | 解析失败、无权限、已删除状态一致 |
| M13-T05 | 统一对象卡片组件 | IM、文档、搜索共用卡片 | 不再散落重复卡片逻辑 |
| M13-T06 | 对象生命周期状态统一 | active、deleted、forbidden、missing | 前后端状态展示一致 |
| M13-T07 | 增加集成测试 | 链接解析和权限卡片测试 | 不泄露无权限对象敏感字段 |
| M13-T08 | 输出对象模型开发规范 | 文档规范 | 新模块按规范接入 |

目标：把各模块散落的权限判断收束为平台级决策能力。

| 编号 | 任务 | 交付物 | 验收标准 |

| M14-T03 | 文档模块接入统一权限 | 文档权限判断迁移 | 文档详情、卡片、搜索结果一致 |
| M14-T04 | 项目模块接入统一权限 | 项目和 issue 权限迁移 | 项目成员外不可越权 |
| M14-T05 | 表格模块接入统一权限 | base/table/record 权限迁移 | 表格记录卡片权限正确 |

## 6. M15 统一实时通道

| M15-T01 | 梳理现有 WebSocket 事件 | 事件清单 | 明确事件来源和消费者 |

| M15-T03 | 支持订阅模型 | 用户、会话、对象订阅 | 不向无权限用户推事件 |

| M15-T08 | 补断线和重复事件测试 | 自动测试 | 断线补拉和去重稳定 |

## 10. 工作循环要求

每轮执行遵循：

1. 读取现有代码和文档，确认任务真实状态。
2. 明确本轮任务范围和验收标准。
3. 小步修改，避免跨模块无关重构。
4. 执行对应测试、构建和质量门禁。
5. 更新文档、脚本或测试用例，沉淀可复用流程。
6. 输出完成项、未完成项、风险和下一步入口。

M12 之后的关键原则：先平台化，再复杂化。不要在统一对象、权限、实时、搜索缺失时继续堆大型编辑器或多端工程。
