# 当前路线图

本文档是唯一活跃路线图。已完成的 M25-M30 计划已归档到 `docs/99-archive/superseded-roadmaps/m25-m30-completed-roadmap.md`。

## 当前阶段判断

系统已经完成基础模块、核心协作体验打磨和单节点交付运维收口。下一阶段不急于继续横向铺新模块，也不直接进入团队真实内测；当前更需要建立一套可重复、可断言的深度协同仿真基线，用固定角色、固定项目、固定剧本暴露真实协同短板。

当前执行重点：

1. M31 建立“多角色 Agent 协同仿真与分支测试基线”。
2. 通过 10 人、5 项目、多分支需求/BUG 流转，验证 IM、项目、文档、Base、通知、权限、审计和搜索的联动质量。
3. 将文档团队空间、树形目录、文档嵌入 Base 表格作为 M31 明确暴露的缺口，后续再拆 M32/M33 实现。

## M31 多角色 Agent 协同仿真与分支测试基线

目标不是生成更多测试数据，而是建立一套可重复运行的团队协同仿真系统，用固定 10 人角色、5 个项目场景、多分支需求/BUG 流程，持续验证 IM、项目、文档、Base、通知、权限、审计、搜索之间的协同质量。

| 编号 | 任务 | 交付物 | 验收标准 | 状态 |
| --- | --- | --- | --- | --- |
| M31-T01 | 固定 10 人多角色测试组织 | 账号、角色、权限矩阵 | 每个账号有明确身份、权限、项目参与关系 | 计划中 |
| M31-T02 | 设计 5 个项目协同场景 | 场景清单、项目背景、模块覆盖矩阵 | 每个项目至少覆盖 IM + 项目 + 文档/Base/通知 中 3 类联动 | 计划中 |
| M31-T03 | 设计多角色 Agent 行为模型 | Director、Role、Verifier、Defect Analyst 规则 | Agent 不自由发散，必须按剧本推进和断言 | 计划中 |
| M31-T04 | 设计需求流转分支剧本 | 需求确认、补充、变更、延期、取消等分支 | 每条分支有状态、通知、审计、文档版本断言 | 计划中 |
| M31-T05 | 设计 BUG 流转分支剧本 | 有效、重复、无法复现、信息不足、验证失败、验证通过 | 支持打回、二次修复、关闭、关联已有 BUG | 计划中 |
| M31-T06 | 实现可重复仿真数据准备脚本 | 固定账号、项目、会话、文档、Base、事项、权限 | 数据少而完整，可重复重置，不生成无意义噪音 | 计划中 |
| M31-T07 | 实现自动验证与浏览器深度冒烟 | API 断言 + 浏览器路径验证 | 校验状态、通知、权限、审计、搜索、IM 卡片、页面可见性 | 计划中 |
| M31-T08 | 输出 M31 仿真报告与缺陷清单 | 执行报告、缺陷分类、后续里程碑建议 | 缺陷可追踪，能反推 M32/M33 的真实优先级 | 计划中 |

## M31 分支覆盖

需求分支：

- 正常确认并拆任务。
- 需求信息不足，打回产品补充。
- 需求中途变更，产生文档版本和 IM 讨论。
- 需求延期，通知相关成员。
- 需求取消，状态、审计、通知正确。

BUG 分支：

- BUG 有效，进入修复。
- BUG 信息不足，打回补充。
- BUG 重复，关联已有 BUG。
- BUG 无法复现，关闭或待确认。
- 修复后验证失败，打回开发。
- 二次修复后验证通过并关闭。
- 无权限用户访问被拦截。
